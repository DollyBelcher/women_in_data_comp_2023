Description:
Abstract:
This data publication contains urban tree inventory data for 929,823 street trees that were collected from 2006 to 2013 in 49 California cities. Fifty six urban tree inventories were obtained from various sources for California cities across five climate zones. The five climate zones were based largely on aggregation of Sunset National Garden Book's 45 climate zones. Forty-nine of the inventories fit the required criteria of (1) included all publicly managed trees, (2) contained data for each tree on species and diameter at breast height (dbh) and (3) was conducted after 2005. Tree data were prepared for entry into i-Tree Streets by deleting unnecessary data, matching species to those in the i-Tree database, and establishing dbh size classes. Data included in this publication include tree location (city, street name and number), diameter at breast height, species name and/or species code, and tree type.
Purpose:
These data were used to calculate street tree stocking levels, species abundance, size diversity, function and value, which can be used to determine trends in tree number and density, identify priority investments and create baseline data against which the efficacy of future practices can be evaluated.

Classification_System_Modifications:
ITIS was used as our main classificiation system. Unfortunately not all of the species included in this data publication had taxonomy in ITIS. Here is a description of what we did:

1. Used ITIS to define taxonomy for all species as available

2. For some species, ITIS suggested a more acceptable name - and that taxonomy was provided instead (but note that the data files were not changed and the original species name is still provided there).

3. For all of the other species needing taxonomy we obtained the taxonomic hierarchy from other sources. See \Supplements\species_alternate_taxonomic_source.csv for a list of those species, the taxonomic source, and the taxonomic hierarchy. Unfortunately there were a few species for which we could not obtain the taxonomy - and that is noted in this file.

Process_Step:
Process_Description:
Inventories were acquired from city employees, West Coast Arborist (WCA), and CAL FIRE. An employee of each city or organization was contacted and datasets sent through email. Each inventory was checked to determine if it met the required criteria of (1) included all publicly managed trees, (2) contained data for each tree on species and diameter at breast height (dbh) and (3) was conducted after 2005. Acceptable inventories were "cleaned" so that species names and codes were consistent and dbh classes were standardized.

The data obtained were collected by different people and for different purposes. Therefore, some of the data were unnecessary for the needs of this project and were deleted. Each data set was prepared for entry into i-Tree Streets by matching species to those in the i-Tree Streets database, establishing dbh size classes, and when necessary assigning a tree type to each tree. As some of the inventories contained synonyms of the species names that are recoginzed by the i-Tree Streets software those synonyms needed to be changed. Since the inventories contained a range for dbh thoses ranges were standardized across the inventories. Six dbh range categories were created and each assigned a number 1-6.

For more details see McPherson et al. 2016.

McPherson, E. Gregory; van Doorn, Natalie; de Goede, John. 2016. Structure, function and value of street trees in California, USA. Urban Forestry & Urban Greening 17: 104-115. https://doi.org/10.1016/j.ufug.2016.03.013

Process_Description:
The U.S. was divided into sixteen national climate zones by aggregation of 45 Sunset climate zones (Brenzel 1997). Sunset zones were aggregated based on factors that influence plant distribution, such as length of growing season and minimum temperature, as well as building energy use patterns (i.e., number of days with highs of 32°Celsius [90°Fahrenheit] or higher). Also, ecoregions developed by Bailey (2002) and Breckle (1999) were consulted to delineate climate zone boundaries.

This data publication contains 50 data files and 1 supplemental file. Below you will find a detailed description of these files and their contents.

DATA FILES

\DATA\Street_species_codes.csv: Comma-delimited ASCII text file containing a list of i-Tree Streets species, codes, scientific names, and common names for trees found in each climate zone. These species codes are used in the inventory data. Below is a list of variables included in this file.

ClimateZone = Name of climate zone (InlandEmpire = Inland Empire, InlandValley = Inland Valley, NorCalCoast = Northern California Coast, SoCalCoast = Southern California Coast, and SWDistrict = Southwest Desert; see Brenzel 1997).

SpeciesCode = 4 to 6 letter code consisting of the first two letters of the genus name and the first two letters of the species name. Some four-letter codes are followed by a number to distinguish two species with the same code.

ScientificName = Scientific name of species.

CommonName = Common name of species.

TreeType = 3 letter code where the first two letters refer to life form (BD=broadleaf deciduous, BE=broadleaf evergreen, CE=coniferous evergreen, PE=palm evergreen) and the third letter is mature height (S=small which is < 8 meters, M=medium which is 8-15 meters, and L=large which is > 15 meters).

SppValueAssignment = Species assigned value, which was used when a species in the street tree inventory was not one for which growth equations were derived. Such a species was assigned a species of the same genus with similar growth characteristics for which growth equations were derived. For an inventoried species for which there were no matches based on genus, a species was assigned based on the nearest mature height and crown diameter.


\DATA\*CLIMATE ZONE*\*CITYNAME*_Inventory.csv: 49 comma-delimited ASCII text files containing raw urban tree inventory data for the specified city. These street tree data were collected between 2006 and 2013. At a minimum the file includes, a unique ID for each tree, species code, diameter at breast height, street name and address. Optional variables include, botanic and common name, and tree type. The data files are provided in a folder based on the *CLIMATE ZONE* (InlandEmpire, InlandValley, NorCalCoast, SoCalCoast, SWDistrict) that city is in. Below is a list of variables included in most of these files.

Id.x = Number assigned to each tree in the inventory by city.

SpsCode = 4 to 6 letter code consisting of the first two letters of the genus name and the first two letters of the species name. Some four-letter codes are followed by a number to distinguish two species with the same code. Species names are defined in \Data\Street_species_codes.csv.

DBH = Diameter of tree at breast height (4.5 feet) defined by categories. (1 is 0-6 inches, 2 is 6-12 inches, 3 is 12-18 inches, 4 is 18-24 inches, 5 is 24-30 inches, and 6 is > 30 inches).

StreetName = Name of the street the tree is located on, from inventory.

StreetNumber = Street number of building where tree is located, from inventory.

City = City name where data were collected.
